# 知识便利贴 · LangGraph 文本对话机器人

本文对应示例：**LangGraph → 文本对话机器人**。右侧「知识便利贴」只展示每条的一句话摘要；下文为展开说明。

本示例尽量用 **LangGraph** 管会话、流式与压缩；模型用 `ChatQwen`；压缩算法复用 `chat_compress`。

```text
Human 入图
  → ensure_system
  → assistant（trim_messages + ChatQwen.invoke）
  → compress（满 20 轮混合压缩）
  → graph.stream(stream_mode="messages") 流出思考 / 回答
  → SqliteSaver 落库
  → finalize_after_turn（读 did_compress）
```

---

## 1. ChatState + ensure_system

自定义状态：`messages`（`add_messages`）+ `did_compress`。  
`ensure_system` 在尚无系统提示时写成 `[System, ...]`，让 **system 进 checkpoint**。

## 2. SqliteSaver + WAL

按 `thread_id` 持久化；连接开启 `PRAGMA journal_mode=WAL`。

## 3. graph.stream(stream_mode="messages")

```python
graph.stream(
    {"messages": [HumanMessage(...)]},
    {"configurable": {"thread_id": "...", "enable_thinking": True}},
    stream_mode="messages",
)
```

LangGraph 转发 `assistant` 内 `ChatQwen.invoke` 的 token；思考在 `additional_kwargs["reasoning_content"]`。

## 4. compress 节点

满 20 轮时在图内混合压缩并 `RemoveMessage(REMOVE_ALL)` 写回；保留头部 `SystemMessage`，并置 `did_compress=True`。  
`finalize_after_turn` 只负责读状态、拼 UI，不再单独跑压缩。

## 5. 终止：Event + checkpoint 时间旅行

- UI 点「终止」→ `Event` 打断 stream 消费  
- 发送前记下 `checkpoint_id`；取消后从该检查点读 values，再 `update_state` fork 成新 tip，丢弃本轮  

比手写「消息列表快照」更贴 LangGraph 持久化模型。

## 6. 图结构

`START → ensure_system → assistant → compress → END`。
