# 知识便利贴 · 同步与异步

本文对应示例：**Python → 同步与异步**。右侧「知识便利贴」只展示每条的一句话摘要；下文为展开说明。

---

## 1. asyncio 模块

`import asyncio` 后可用协程与事件循环写并发。适合大量**等待型**（IO）任务：等待时把控制权交回事件循环，去推进其它协程。

## 2. async def / await

- `async def`：定义协程函数（调用后得到协程对象，不会立刻跑完）
- `await`：在协程内挂起，等待某个可等待对象完成；挂起期间事件循环可调度其它任务

## 3. asyncio.sleep 与 time.sleep

| 写法 | 行为 |
| --- | --- |
| `time.sleep(...)` | 堵住**当前线程**，别的事也干不了 |
| `await asyncio.sleep(...)` | 只挂起**当前协程**，事件循环仍可跑其它协程 |

本示例用这两种 sleep 对比「同步串行」与「异步重叠等待」。

## 4. asyncio.gather 并发执行

`asyncio.gather(...)` 是常用的并发入口：把多个协程一起调度，等它们都结束后汇总结果。多个 `asyncio.sleep` 的等待可以重叠，总耗时接近**最慢的那个**，而不是相加。

## 5. * 解包不确定个数的任务

任务个数不固定时，不要手写一长串参数，而是：

```python
await asyncio.gather(*(fetch_async(i) for i in range(TASK_COUNT)), ...)
```

`*` 把可迭代对象拆成位置参数，再传给 `gather`。

## 6. return_exceptions=True

默认某个子任务抛错时，`gather` 会把异常抛给调用方，并可能影响对其它任务的等待方式。  
加上 `return_exceptions=True` 后：某个异步任务失败**不打断全局**，异常作为对应位置的「结果」返回，便于一并检查处理。

## 7. asyncio.run 启动入口

`asyncio.run(run_async())` 负责：创建事件循环 → 跑完入口协程 → 清理。适合在普通脚本的 `if __name__ == "__main__"` 里启动异步主流程。

## 8. 异步调用链路

本示例的典型调用链：

```text
asyncio.run()
  └─ async def run_async
        └─ await asyncio.gather(...)
              └─ async def fetch_async
                    └─ await asyncio.sleep(...)
```

记住：外层用 `asyncio.run` 启动；中间用 `gather` 并发；最里层用 `await` 真正挂起等待。
